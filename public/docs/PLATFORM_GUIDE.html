Mada Graphite 平台总览与操作手册

B2B 石墨交易平台 — 公开官网、买家 / 卖家交易流程与管理后台说明

本手册面向业务与运营人员，说明各页面功能与日常操作步骤。界面按钮与菜单名称保留英文原文，便于对照实际系统。

1. 平台概览 ↑ 目录

Mada Graphite 是一个 B2B 石墨交易平台，连接全球买家与马达加斯加本地卖家， 主要供应方为自有矿场的 Graphite Energy Inc.。平台整合 AI 智能咨询、市场情报与线上 B2B 交易， 涵盖从询价、议价、合约签署、多阶段付款到出货清关的完整流程。

用户角色

角色说明能做什么
访客未登录用户浏览公开页面、使用 AI 咨询（仅知识问答）
买家已验证买家询价、下单、付款、确认收货、站内消息
卖家已验证卖家上架商品、回应询价、起草合约、更新出货、审核付款
管理员平台运维用户管理、品类管理、订单监管、付款覆审、新闻与内容管理、争议介入
超级管理员平台负责人管理其他管理员、敏感平台设置

核心交易流程

买卖双方先通过「询价 → 报价 → 议价」敲定条件；买家接受报价后生成订单，进入合约签署与履约阶段。 付款按合约约定的多期分期进行，与订单出货进度并行——未付清的分期只影响订单能否标记为「已完成」，不会卡住出货等其他环节。

第一阶段：询价议价
买家提交询价 → 卖家回复报价 → 双方来回议价 → 买家接受报价

第二阶段：合约与生产
待签合约 → 合约已签 → 生产中 → 备货完成 → 已出货 → 运输中 → 已到港 → 清关完成 → 订单完成
（任意阶段可发起争议或取消订单）

付款（与上述并行）
待付款 → 到期应付款 → 待审核 → 已付款（全部付清后订单才能完成）

3. 账号与通用功能 ↑ 目录

3.1 注册与登录

  • 邮箱注册（Register 页面）：填写邮箱、密码、选择角色（买家 / 卖家）、公司名与国家。系统发送验证邮件，点击链接后账号激活。
  • 邮箱验证（Verify email 页面）：若未收到验证信，可在此重新发送。
  • Google 登录：点击「Continue with Google」一键登录，默认角色为买家，公司资料可日后补齐。
  • 忘记 / 重置密码（Forgot password → Reset password）：通过邮件链接设置新密码。原本仅用 Google 登录的账号，也可借此绑定邮箱密码，两种方式登录同一账号。

3.2 登录后的通用页面

菜单 / 页面功能
Dashboard（仪表盘）根据您的角色显示待办事项（待回复询价、待付款、待签合约等）、进行中订单与快捷入口。管理员另显示「Go to Admin Console」。
Settings（设置）编辑个人与公司资料、头像、语言切换、KYC 入口。询价、上架、付款前须填写完整的公司名称与国家。
KYC（身份验证）手机验证、证件上传，共四个验证等级（0–3）。部分操作有最低等级要求。
Messages（消息）与交易对手的一对一站内消息。每对买家 + 卖家各有一条对话线程，可从市场、询价或订单页点击「Message」进入。
资料不完整时：若公司名称或国家未填写，系统会在您尝试询价、上架或付款时提示，并引导前往 Settings 补全资料。
语言：登录后的仪表盘与主要公开页面支持英文 / 简体中文切换（在 Settings 或导航栏切换）。 合约正文、邮件通知、管理后台界面与新闻内容目前仍以英文为主。

4. 买家端 ↑ 目录

买家 买家的核心任务：找货 → 询价 → 议价 → 签约 → 分期付款 → 收货。

4.1 浏览与询价

功能介绍：在市场浏览上架商品，查看规格与卖家信息，并提交询价。

  1. 浏览市场
    侧边栏进入 Market，以卡片浏览所有在售商品，显示规格标签、最小起订量（MOQ）与图片。
  2. 查看单品
    点击商品卡片进入详情页，查看图片、规格参数、卖家信息与可出货时间。
  3. 提交询价
    点击「Submit Inquiry」，填写采购数量、目标价格、目的地与备注。系统会检查起订量、资料完整性与 KYC 等级，通过后生成询价单。
  4. 跟踪询价
    侧边栏进入 Inquiries，在 Sent（已发送） 分页查看询价状态。
  5. 与卖家沟通
    可在商品详情页点击「Message」私讯卖家，或在 Messages 中查看对话。

4.2 议价与下单

功能介绍：卖家回复报价后，买家可在询价详情页审阅、议价或接受。接受报价即自动生成订单。

  1. 审阅报价
    进入询价详情页，查看报价时间线（规格、价格、贸易条款 Incoterm、有效期）。
  2. 议价或接受
    可选择 Counter（还价）、Decline（拒绝） 或 Accept（接受）。
  3. 接受报价
    点击 Accept 后系统自动创建订单，贸易条款锁定，并跳转至订单详情页，进入「待签合约」阶段。
提出报价的一方不能接受自己的报价；接受动作须由收到报价的一方执行。

4.3 合约签署

功能介绍：卖家起草合约后，买家在 Contract 分页预览、核准或退回，并上传签名扫描件。

  1. 预览合约
    在订单详情页打开 Contract 分页，查看合约内容与付款排程表，可点击打印输出 PDF。
  2. 核准或退回
    点击 Approve（核准） 表示同意合约内容；若需修改，点击 Reject（退回） 并填写原因，卖家将重新起草。
  3. 上传签名扫描
    打印合约并签名后，上传买方签名件（支持图片或 PDF）。
  4. 完成签约
    买卖双方均上传签名且买方核准后，订单自动进入「生产中」阶段。

4.4 付款

功能介绍：按合约约定的多期分期付款，与订单出货进度并行。卖家为付款主审人。

  1. 查看排程
    在订单 Payment 分页查看各期付款状态；进度条右上角显示「Payments X/Y paid」（已付 X 期 / 共 Y 期）。
  2. 提交付款
    对「到期应付款」或「逾期」的分期，填写付款信息并提交；对尚未到期的分期也可选择「Pay Early（提前付款）」。
  3. 付款方式
    支持 USDT（TRC20 / ERC20）、USDI、MUP、银行转账。链上方式填写交易哈希；银行转账等方式上传付款凭证图片。
  4. 等待审核
    提交后进入「待审核」状态，由卖家确认；若被拒绝，需修正后重新提交。

4.5 出货追踪与收尾

  1. 追踪物流
    在 Shipment 分页查看提单号（B/L）、船名、货柜号、预计 / 实际到港时间等。
  2. 文件中心
    在 Documents 分页查看或上传各类贸易文件（合约、发票、物流、检验、清关、付款凭证等，共 13 类）。
  3. 到港与清关
    买卖双方任一方可确认「已到港」；买家确认「清关完成」后订单进入清关完成阶段。
  4. 完成订单
    清关时若全部付款期数已付清，订单自动标记为「已完成」；若仍有未付分期，须待最后一笔付款确认后才能完成。

5. 卖家端 ↑ 目录

卖家 卖家的核心任务：上架 → 回应询价 → 起草合约 → 生产出货 → 审核付款。

5.1 上架管理

功能介绍：创建并维护在售商品，含规格、起订量、图片与上下架状态。

  1. 新建商品
    侧边栏进入 My Listings → 点击「New Listing」，选择产品品类，填写标题、规格、数量 / 单位、出货地、可出货时间、单价 / 币种、贸易条款、备注与图片。提交后立即出现在市场。
  2. 规格与起订量
    按品类填写结构化规格（产品类型、网目、固定碳范围等）；Custom Grade 可自定义网目。可设置最小起订量（MOQ），也可使用「Generate title」自动生成标题。
  3. 上传图片
    支持上传新图片或从既往图库中选用，系统自动压缩优化。
  4. 管理商品
    在 My Listings 列表中可执行：Edit（编辑）、Pause（暂停）、Resume（恢复）、Sold-out（售罄）、Delete（删除）。已有订单关联的商品无法删除。

5.2 报价与议价

  1. 收到询价
    在 Inquiries 的 Received（已收到） 分页查看，并会收到邮件 / 短信通知（若已开启）。
  2. 发送报价
    进入询价详情页，填写正式报价（规格、价格、贸易条款、有效期，默认 14 天）；或对 pending 询价使用快速接受生成默认报价。
  3. 议价来回
    可发送还价或拒绝询价 / 报价。
  4. 等待买家接受
    买家 Accept 后，订单自动创建并进入「待签合约」阶段。

5.3 合约起草

  1. 起草合约
    在订单 Contract 分页，选择贸易条款（FOB / CFR / CIF），系统按模板生成合约内容。
  2. 设定付款排程
    拆分 1–10 期付款（预付款 / 常规款 / 尾款，总和须为 100%），设定各期比例与触发条件。
  3. 上传签名扫描
    打印合约并签名后，上传卖方签名件。
  4. 买家退回时
    若买家 Reject，根据退回原因修改后重新起草（版本号递增）。

5.4 生产与出货

  1. 生产中
    签约完成后通常自动进入「生产中」；可在订单概览页操作各阶段。
  2. 备货完成
    点击「Ready to Ship（备货完成）」更新状态。
  3. 出货
    在 Shipment 分页填写提单号、船名 / IMO、货柜号、开船 / 到港时间，确认出货；可选上传提单扫描与检验报告（COA / SGS）。
  4. 在途与到港
    依次更新「运输中」「已到港」状态；到港可由买卖双方任一方确认。

5.5 付款审核（卖家为主审）

  1. 审核付款
    买家提交付款后进入「待审核」，卖家在 Payment 分页点击 Verify（通过）或 Reject（拒绝）。通过后该期标记为「已付款」；拒绝后买家需重新提交。
  2. 付款节点确认
    Payment 分页提供若干「里程碑」按钮（如生产前、出货前、装船前等），用于触发对应期数的付款到期。部分节点会随订单状态自动触发，部分需手动点击确认。
付款审核以卖家为主；若卖家无法及时处理或发生争议，管理员可在管理后台的 Payments 页面介入覆审。

6. 管理后台 ↑ 目录

管理员 超级管理员 管理后台仅管理员可进入，涵盖平台运维、交易监管与内容管理。登录后在 Dashboard 点击「Go to Admin Console」进入。

6.1 仪表盘总览

入口：Admin Dashboard

  • 统计卡片：总用户数、总订单数、争议订单数、待审付款数。
  • 优先待办（Priority Actions）：争议订单、待审付款、需处理用户（KYC 待审 / 账号待激活 / 资料不完整）。
  • 侧边栏数字提醒：Users、Payments、Orders disputed 等待办数量。

6.2 用户与 KYC 管理

入口：Users

  1. 浏览与筛选
    查看全平台用户（邮箱、公司、角色、状态、KYC 等级），可按需处理 / 角色 / 状态 / KYC 筛选与搜索。
  2. 冻结 / 解冻
    对违规或风险用户执行 Freeze（冻结），冻结后无法正常使用平台；解冻后恢复。
  3. 变更角色
    可将用户设为买家、卖家或管理员。将用户提升为管理员仅超级管理员可执行。
  4. KYC 审核
    打开用户 KYC 对话框，查看上传证件与手机验证状态，核准证件后自动升级等级，或手动设定等级（0–3）并填写备注。

6.3 订单、付款与文件监管

入口能做什么
Orders（订单列表） 查看全平台所有订单（订单号、买卖双方、金额、状态）。
Order Detail（订单详情） 查看订单全貌：进度、合约签名状态、付款排程、文件、时间线。 可进行管理员介入：Force Transition（强制切换状态，用于争议解决）、发起争议、取消订单（出货前）。 可查看该订单的操作记录（最近 20 条）。
Payments（付款覆审） 查看待审核与历史付款记录，执行 Verify / Reject，作为卖家无法及时处理时的备用审核。买家提交付款时管理员会收到邮件通知。

文件核验：管理员可在订单详情标记贸易文件为「已核验」，也可删除任何文件。普通用户仅能在上传后 1 小时内删除自己未核验的文件。

6.4 内容管理与平台设置

入口能做什么
Categories（产品品类）新增 / 编辑 / 停用产品品类及规格模板（产品类型、网目、固定碳、水分等字段定义）。
News（新闻管理）管理新闻文章：待审 / 已发布 / 已拒绝 / 草稿。可手动撰写、导入候选文章、审核发布 / 下架，以及翻译为多语言版本。
Mine Photos（矿场照片）管理矿场照片分类与图片，设定分类封面，控制是否在公开页面显示。
Partners（合作伙伴）管理首页合作伙伴 logo 与链接，控制排序与是否发布。
Settings（平台设置）邮件服务测试（Send test email）、KYC 门槛设定（询价 / 上架最低等级）、短信通知开关。
说明：管理后台目前没有独立的聊天管理页面与 AI 对话记录浏览界面； 新闻不会全自动爬取，需管理员审核后发布；平台不涉及区块链钱包与自动结算。
