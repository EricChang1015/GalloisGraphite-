For the full coding rules, architectural decisions, domain model, and UI/UX

## Top rules (TL;DR)

1. Server Components by default, `"use client"` only when interactivity is needed.
2. Mutations use Server Actions in `src/actions/*`, not API routes.
3. Forms use `react-hook-form` + `zod`, rendered with shadcn `Form`.
4. DB access uses `@supabase/ssr` clients in `src/lib/supabase/`.
5. Tailwind v4 + shadcn (style: `base-nova`, base library: `@base-ui/react`).
6. No blockchain / wallet SDKs — payments are manually verified by admins.
7. Never expose `SUPABASE_SERVICE_ROLE_KEY` outside server-only modules.
8. **Schema changes** must go through `supabase/migrations/NNN_*.sql` and be
   applied with `npm run db:migrate` (Supabase Management API, no DB password).
   Never instruct the user to copy/paste SQL into the Dashboard. See
   [`.cursor/rules/migrations.mdc`](./.cursor/rules/migrations.mdc).
9. **Build gate**: `npm run build` must exit 0 before every `git commit`. See
   [`.cursor/rules/git.mdc`](./.cursor/rules/git.mdc).
10. **Dev server lifecycle**: stop the dev server when you're done with it.
    If port 3000 is already in use, *stop / kill the existing process* before
    starting a new one — do **not** fall back to another port. Stale 3001/3002
    instances leak DB sockets and confuse smoke tests. On Windows:
    `Get-NetTCPConnection -LocalPort 3000 | Select OwningProcess` then
    `Stop-Process -Id <pid> -Force`.

## Cursor Cloud specific instructions

### Environment

- Node.js v22 is pre-installed. The project uses **npm** (lockfile: `package-lock.json`).
- All required secrets (`NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `POE_*`, `SMTP_*`, etc.) are injected as environment variables.
- **`.env.local` must be generated** before running `npm run dev` or `npm run build`, because Next.js reads env vars from that file at startup rather than inheriting from the shell in all contexts. Generate it with:
  ```bash
  env | grep -E "^(NEXT_PUBLIC_SUPABASE_URL|NEXT_PUBLIC_SUPABASE_ANON_KEY|SUPABASE_SERVICE_ROLE_KEY|SUPABASE_ACCESS_TOKEN|POE_API_KEY|POE_MODEL|POE_BASE_URL|SMTP_HOST|SMTP_PORT|SMTP_USER|SMTP_PASS|SMTP_SECURE|EMAIL_FROM_DOMAIN|EMAIL_FROM_ADDRESS|EMAIL_FROM_NAME|SMS_BASE_URL|SMS_APP_ID|NEXT_PUBLIC_APP_URL|NEXT_PUBLIC_APP_NAME|ADMIN_EMAIL|PLATFORM_USDT_TRC20_ADDRESS|PLATFORM_USDT_ERC20_ADDRESS|PLATFORM_BANK_INFO|NEXT_PUBLIC_ENABLE_SELLER_SELF_SIGNUP|NEXT_PUBLIC_ENABLE_AI_CHAT|PLATFORM_USDI_ADDRESS|PLATFORM_MUP_ADDRESS)=" > .env.local
  ```

### Running the dev server

- `npm run dev` starts the Next.js 16 dev server on port 3000 (Turbopack).
- On Linux (Cloud VM), kill stale port 3000 with: `lsof -ti:3000 | xargs kill -9 2>/dev/null` before restarting.

### Testing notes

- There are no automated test suites (no Jest/Vitest/Playwright). Validation is done via `npm run build` (TypeScript + Next.js compilation) and manual browser testing.
- Pre-existing lint errors in `docs/oldSite/` (legacy JS files) and `src/types/database.ts` (binary/autogenerated) are expected and safe to ignore.
- Test accounts are documented in `docs/TESTING.md`. All use password `a1234567`.

### Gotchas

- The homepage and public marketing pages (`/about`, `/products`, `/news`, etc.) require Supabase connectivity. If the Supabase URL/key are missing or invalid, these pages will 500 while auth pages (`/login`, `/register`) still render fine.
- `src/proxy.ts` is the Next.js 16 equivalent of `middleware.ts` (renamed in v16).
- The project uses **Tailwind CSS v4** — there is no `tailwind.config.js`. Theme tokens are in `src/app/globals.css`.
